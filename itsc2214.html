  • JUnit

    How to assert exceptions in JUnit 4 with the expected attribute.

    Useful if don't know or need a refresher on how to test exceptions for the projects using junit 4. While you might have junit 5 locally on your machine, web-cat doesn't support it so you won't be able to use assertThrows. Using this will allow you to use the expected attribute like this: @Test(expected = Exception.class). Make sure to not use a try/catch in your test case, as that will result in the test failing since the exception will be caught.

  • Web-CAT & Extension Errors

    How to fix "could not find submission url"

    basically means it can't find the url to upload to Web-CAT. That is one of the settings in their settings.json in the profile. It should have this:
    "itsc2214.uploadURL": "https://web-cat.cs.vt.edu/Web-CAT/WebObjects/Web-CAT.woa/wa/assignments/eclipse?institution=UNCC&staff=true",
    but who knows why it doesn't. Profile not installed? Not selected? Using old plugin? Simple solution: zip 2 files (java and test) and go to Web-CAT and submit by hand.
